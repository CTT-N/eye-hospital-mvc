Design — Conversion Pattern

How each HTML page becomes a live JSP with real data

Step 1 — Copy static assets into webapp

File moves (one-time)
fe/css/ → webapp/static/css/
fe/js/ → webapp/static/js/
fe/images/ → webapp/static/images/

Then in every JSP, asset paths use the context path:
<link href="${pageContext.request.contextPath}/static/css/variables.css">

Step 2 — Controller loads data, forwards to JSP

Example: PatientAppointmentController.java (doGet)
PatientDAO dao = new PatientDAO();
int patientId = ((Patient) session.getAttribute("user")).getPatientId();
List<Appointment> appointments = dao.getAppointmentsByPatient(patientId);

req.setAttribute("appointments", appointments);
req.getRequestDispatcher("/views/patient/patient-appointments.jsp").forward(req, resp);

Step 3 — JSP renders the data with JSTL

patient-appointments.jsp — table body (replaces hardcoded HTML rows)
<%@ taglib uri="jakarta.tags.core" prefix="c" %>
...
<c:forEach var="appt" items="${appointments}">
  <tr data-status="${appt.status}">
    <td>${appt.appointmentDate}</td>
    <td>${appt.doctorName}</td>
    <td><span class="badge-status">${appt.status}</span></td>
  </tr>
</c:forEach>

Step 4 — Forms post to correct servlet URLs

Before (broken)
<form action="LoginServlet">
<a href="dashboard.html">
<a href="../book-appointment.html">
After (correct)
<form action="${pageContext.request.contextPath}/auth/login">
<a href="${pageContext.request.contextPath}/patient/dashboard">
<a href="${pageContext.request.contextPath}/patient/appointments">
